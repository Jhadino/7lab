Пользовательская история
Как клиент, я хочу заказать еду из ресторана с доставкой на дом, чтобы не готовить и насладиться любимой едой.

Этап 1: Уточнение требований
Клиент (может быть зарегистрированный или нет) хочет заказать еду чтобы ему привезли домой. Он не хочет готовить сам, а хочет поесть то что нравится.

Вопросы которые нужно обсудить:

Как человек будет искать где заказать? Просто смотреть что рядом или искать конкретное?

Можно ли платить картой или только наличными?

Будет ли видно где сейчас курьер с заказом?

Что если привезли не то что заказывал или еда холодная?

Можно ли заказать чтобы привезли к определенному времени, например к приходу с работы?

2. Что должно получиться в итоге (Критерии)
Человек может найти ресторан через поиск или посмотреть список что есть рядом

Когда зашел в ресторан, видно меню с ценами и можно добавить в корзину

В корзине можно указать куда везти и как платить, система должна посчитать сколько всего стоит

После оформления заказа показывается номер заказа и примерно когда привезут

Ресторан должен как-то узнать что ему нужно приготовить заказ

3. Кто будет пользоваться системой
Клиент - тот кто заказывает еду

Ресторан - тот кто готовит

Курьер - тот кто везет заказ

Система - все работает автоматически


User Story Map (Карта пользовательской истории)
Карта истории пользователя
Тема: Заказать еду с доставкой

Как клиент я хочу: заказать еду из ресторана с доставкой

Основные шаги:
Найти где заказать и выбрать еду

Минимально: посмотреть какие рестораны есть, открыть меню, добавить что-то в корзину

Еще можно добавить: фильтры по типу еды, цене, поиск конкретных блюд, рекомендации что раньше заказывал

Заказать и оплатить

Минимально: написать адрес, выбрать как платить, нажать кнопку "заказать"

Еще можно добавить: выбрать точное время доставки, написать комментарий для курьера

Получить заказ и оценить

Минимально: получать смс когда статус меняется (приняли, готовят, везут), забрать у курьера, можно поставить оценку

Еще можно добавить: смотреть на карте где курьер, писать в поддержку если проблемы, получать бонусы за заказы

4. Диаграмма состояний для сущности Заказ
@startuml
title Диаграмма состояний Заказа

[*] --> Created : Клиент оформил заказ
Created --> Confirmed : Ресторан подтвердил
Confirmed --> Preparing : Ресторан начал готовить
Preparing --> ReadyForDelivery : Готов к выдаче курьеру
ReadyForDelivery --> OnTheWay : Курьер принял заказ
OnTheWay --> Delivered : Курьер доставил
OnTheWay --> Problem : Возникли проблемы

Delivered --> [*]
Problem --> Reviewed
Problem --> [*] : Отменен/Возврат

Delivered --> Reviewed : Клиент оставил отзыв
Reviewed --> [*]

note right of Created
  Система уведомляет
  ресторан о новом заказе
end note

note right of OnTheWay
  Клиент может отслеживать
  местоположение курьера
end note
@enduml
